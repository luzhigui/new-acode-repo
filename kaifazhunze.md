// ==================== 一、核心铁律 (9条) ====================
/*
1. 发代码必须经总座确认
   - 任何代码修改，发送前必须等总座确认，禁止擅自改动。
   - 优先发前后对比（旧代码块必须准确 → 新代码块），不要动不动发整个文件。

2. 错误可见，不靠瞎猜
   - 异常必须在页面底部红色面板显示（18-error-capture.js），手机端也能看。

3. 定位优先，修复滞后
   - 先找到根因再动手。同一问题连续两次修不好，立刻停手回最小单元排查。

4. 禁止凭感觉改代码
   - 凡涉战斗规则、数值、Buff，不确定就问，严禁"猜测式修改"。

5. 函数职责单一，看动分离
   - 纯计算、纯判定、纯渲染必须隔离。播放器只消费日志，不参与计算。

6. 数据驱动视图
   - 特效走数据标记（_flash、_blocked、_flyMode 等），renderGrid 统一读取渲染。
   - 禁止特效函数直接操作 DOM。

7. 实验室代码逐行复刻
   - 验证通过的函数原样搬入，不做"等价改写"。特效参数提到文件顶部可调。

8. 交付前自审完整性
   - 所有导出函数有实际实现、文件末尾完整、无注释占位符。
   - 禁止"和之前一样"之类的省略。

9. 精英技能完全配置化
   - 参数全在 01-config.js 的 ELITE_SKILLS，逻辑在 05-elite-skills.js。
   - UI 只读取配置，不硬编码。
*/

// ==================== 二、代码交付标准 (4条) ====================
/*
1. 前后对比优先
   格式："旧代码块：" + 旧代码 + "新代码块：" + 新代码。
   仅大改动或跨文件才发完整文件。

2. 文件头部规范
   // 文件名 - 描述
   // 预估行数/字符, 时间, 版本
   export const VER = '文件名 Vx.x.x';

3. 版本号统一
   格式 VX.X.X，发版前全文件对齐。当前统一 V4.0.0。

4. 大文件分片规范
   超 40000 字符自动分片，头部注明 [第 N/M 片]，结尾附确认提示。
*/

// ==================== 三、经验案例库 (6条) ====================
/*
1. 死机排查：先看红色错误面板，再 F12，逐函数追溯，不凭感觉。
2. 数据污染：找源头改，不靠清理循环打补丁。先问"还有哪些地方能改这个数据？"
3. 特效时序：横幅预告在特效前；暂停期间 isPaused=true 确保 DOM 查询准确。
4. 特效残留：飞撞后必须恢复 opacity/visibility/display/filter/transform 全部样式。
5. 模块合并：变量名唯一化、import 用 [\s\S]*? 跨行匹配、用绝对路径。
6. 合并后 VER 冲突：各模块 VER 重命名为 VER_CONFIG、VER_CORE 等全局变量。
*/

// ==================== 四、工具链速览 (14项) ====================
/*
 1. 错误捕获 (18-error-capture.js) — 页面底部红色面板，手机端可见。
 2. 核心单元测试 (19-test-utils.js) — calcDamage 等核心函数一键跑测。
 3. 战斗日志导出 (08-main.js 战报弹窗) — 排行榜 + 导出 JSON/文本。
 4. 音频管理器 (17-audio.js) — Web Audio 合成音效 + BGM 三态切换。
 5. 音效配置 (01-config.js SFX 段) — 音效路径集中管理。
 6. 精英技能描述 — 参数在 config，UI 只读取，不硬编码。
 7. CSS 动画单行原则 — @keyframes 定义单行，保持项目风格。
 8. 全身体检规则 (20-health-rules.js) — 64 条规则，13 分组，独立文件。
 9. 全身体检中心 (test.html) — 自动 6 关检测 + 答题 + 报告。
10. 开发工具箱 (tools.html) — 文件复制/函数替换/急救包/防战计算/自动战斗。
11. 自动批量战斗 — 一键几百场，关卡+海克斯偏好，历史合并。
12. 急救包 — 扫描经典 Bug（pos=-1 等），一键修复。
13. 防战计算器 — M107 基准阈值表 + 严阵 Buff 后分布验证。
14. 文件复制器 — 大文件自动分片 + 序列发送。
*/
